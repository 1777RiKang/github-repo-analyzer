## 0:00-0:30 开场 — 项目介绍

大家好，我是时航，学号 24211870223。我的课程项目是「GitHub 仓库数据分析系统」。

这个项目能做什么？输入一个 GitHub 用户名，系统自动采集他所有的公开仓库数据，然后从 10 个维度做深度统计分析，生成 10 张专业可视化图表、文字报告和 PDF 报告。还提供了一个交互式 Web 界面，可以浏览 GitHub 热门新项目，也支持两个用户并排对比。项目已部署到 GitHub Pages 静态站点和 Streamlit Cloud。

---

## 0:30-1:30 技术栈与架构

项目用了 7 个 Python 库：

- **requests** — 调用 GitHub REST API v3，支持分页、Token 认证、指数退避重试、429 限流处理
- **pandas** — 数据清洗和分析引擎，做分组聚合、统计计算
- **numpy** — 辅助数值计算，直方图分桶、条件赋值
- **matplotlib** — 生成 10 张专业图表，使用 Agg 后端导出 PNG，支持中文
- **streamlit** — 构建交互式 Web 界面，支持 @st.cache_data 缓存加速
- **wordcloud** — 生成 Topics 标签词云，按星数加权
- **python-docx** — 论文内容写入

项目由 8 个模块组成：main.py CLI 入口，app.py Streamlit Web 界面，github_api.py 数据采集与限流，trending.py 热门发现，analyze.py 10 维分析，visualize.py 图表生成，pdf_export.py PDF 报告导出，build_submission.py 提交打包。含 22 个单元测试。

## 1:30-2:30 核心功能演示

**功能 1：命令行分析**
运行 `python main.py --user torvalds`，系统自动连接 GitHub API 采集仓库数据。生成 10 张图表：
- Stars TOP 排行柱状图
- 编程语言分布饼图
- Stars vs Forks 散点图
- 仓库创建趋势图
- Stars 分布直方图
- 语言平均星数排行
- Topics 标签词云（按星数加权）
- 仓库活跃度评分
- 语言趋势雷达图
- 项目健康度

同时输出包含 10 个维度的文字分析报告和 PDF 报告。所有数据 CSV 缓存，重复运行秒出结果。

**功能 2：Web 交互界面**
打开 Streamlit 界面，三个页签：
- 「热门项目」按每日/每周/每月查看 GitHub 上最火的新仓库，支持 14 种语言筛选
- 「用户分析」输入用户名，一键生成全部图表和数据表，支持 PDF/CSV 下载
- 「对比分析」两个用户并排对比，高亮领先方

**功能 3：线上部署**
- GitHub Pages：`https://1777riKang.github.io/github-repo-analyzer/` — 包含 Microsoft、TheAlgorithms、Linus Torvalds 三个用户的完整分析
- Streamlit Cloud：完整交互功能部署

## 3:30-4:30 遇到的困难和技术亮点

1. **Windows GBK 编码**：命令行输出中文和 emoji 频繁报错，通过 sys.stdout.reconfigure(encoding='utf-8') 解决。

2. **中文 matplotlib 乱码**：默认字体无中文字，图表标题显示方块。通过探测系统字体并配置 font.sans-serif 为微软雅黑解决。

3. **GitHub API 限流**：未认证 60 次/小时。解决方案：Token 认证（5000 次/小时）+ 指数退避重试 + CSV 本地缓存 + Streamlit @st.cache_data 三层缓存架构。

4. **Topics 精确匹配**：str.contains("go") 误匹配 "google"，改用正则 `(?:^|,)keyword(?:,|$)` 精确匹配逗号分隔标签。

5. **自研活跃度评分算法**：综合 push 活跃度(40%)、星数(20%)、Fork 比率(15%)、Issue 管理(15%)、归档状态(10%)五个维度，指数衰减归一化，输出 0-100 分数。

6. **PDF 中文乱码修复**：文字页 monospace 字体无中文字符，改为 sans-serif 后正常。
