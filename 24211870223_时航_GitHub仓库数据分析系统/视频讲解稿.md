**时长：约 5 分钟**

---

## 0:00-0:30 开场 — 项目介绍

大家好，我是时航，学号 24211870223。我的课程项目是「GitHub 仓库数据分析系统」。

这个项目能做什么？简单来说，输入一个 GitHub 用户名，系统自动采集他所有的公开仓库数据，然后从多个维度做统计分析，生成可视化图表和文字报告。还提供了一个交互式 Web 界面，可以浏览 GitHub 每日/每周/每月的热门新项目，也支持两个用户并排对比。

---

## 0:30-1:30 技术栈与架构

项目用了 6 个 Python 库：

- **requests** — 调用 GitHub REST API，支持分页、Token 认证、限流重试
- **pandas** — 数据清洗和分析引擎，做分组聚合、统计计算
- **numpy** — 辅助数值计算，直方图分桶、条件赋值
- **matplotlib** — 生成 7 张专业图表，使用 Agg 后端导出 PNG
- **streamlit** — 构建交互式 Web 界面，支持缓存加速
- **wordcloud** — 生成 Topics 标签词云

项目由 6 个模块组成：main.py 是 CLI 入口，github_api.py 负责数据采集，trending.py 做热门项目发现，analyze.py 做多维分析，visualize.py 做图表生成，app.py 是 Streamlit Web 界面。

---

## 1:30-2:30 核心功能演示

**功能 1：命令行分析**
运行 `python main.py --user torvalds`，系统自动连接 GitHub API，采集 Linus Torvalds 的 11 个公开仓库。生成了 7 张图表：
- Stars TOP 排行柱状图
- 编程语言分布饼图
- Stars vs Forks 散点图
- 仓库创建趋势图
- Stars 分布直方图
- 语言平均星数排行
- Topics 标签词云

同时输出一份包含 7 个维度的文字分析报告。所有数据和图表保存到本地。

**功能 2：Web 热门项目发现**
打开 Streamlit 界面，在「热门项目」页签可以按 每日/每周/每月 查看 GitHub 上最火的新仓库，支持按语言筛选，结果以卡片式布局展示，可下载 CSV。

**功能 3：对比分析**
选择「对比分析」，输入两个用户（比如 torvalds 和 microsoft），并排展示所有关键指标，领先的一方绿色高亮。语言分布、Stars 分布、词云都左右对比展示。

## 2:30-3:30 工作量说明

这个项目从零开始开发，经历了三个版本迭代：

- **第一版**：B 站 UP 主视频分析。花了两天时间研究 B 站的 WBI 签名算法——包括逆向 64 位置的排列表、处理 URL 编码差异。最终因为调试请求过于频繁被 B 站封了 IP，这个方向放弃了。

- **第二版**：尝试用 crawl4ai + Playwright 爬取知乎。知乎的反爬策略非常激进，完全不登录的情况下，Playwright 也拿不到动态内容。确认不可行后放弃。

- **第三版（最终版）**：GitHub 数据分析。GitHub 的 API 稳定、文档完善、无签名，是理想的数据源。核心功能两天完成，后续追加了 Streamlit Web 界面、热门发现模块、对比分析功能和词云。

代码总量约 800 行，严格遵守 PEP8 规范，每个模块有完整的 docstring。

## 3:30-4:30 遇到的困难

除了前面提到的平台切换，还有几个技术难点：

1. **Windows GBK 编码**：命令行输出中文和 emoji 时频繁报错，通过在入口文件加 `sys.stdout.reconfigure(encoding='utf-8')` 解决。

2. **小数据集崩溃**：TOP 10 图表硬编码了 10，当用户只有 8 个仓库时 matplotlib 报 shape mismatch。改为 `n = min(len(df), 10)` 动态适配。

3. **GitHub 限流**：未认证每小时只能 60 次请求。解决方案是支持 Token 认证（5000 次/小时）+ CSV 本地缓存 + Streamlit 的 `@st.cache_data` 装饰器。

4. **github.com 不可访问**：本地网络无法访问 GitHub 网页，于是 trending 功能从页面爬虫改为用 Search API 模拟，反而更稳定。
